# Architecture Decision Records (ADR)

이 디렉토리는 CursorFlow의 주요 설계 결정 사항을 기록합니다.

## ADR이란?
ADR(Architecture Decision Record)은 소프트웨어 아키텍처의 중요한 결정을 설명하고 그 맥락을 기록하는 짧은 텍스트 파일입니다.

## 작성 규칙
1. **형식**: `docs/ADR/NNNN-title.md` (NNNN은 0001부터 시작하는 일련번호)
2. **상태**: 
   - `Proposed`: 제안됨
   - `Accepted`: 승인됨
   - `Deprecated`: 더 이상 사용되지 않음
   - `Superseded`: 다른 ADR로 대체됨
3. **필수 항목**:
   - Title
   - Date
   - Status
   - Context (배경)
   - Decision (결정사항)
   - Consequences (결과/영향)
